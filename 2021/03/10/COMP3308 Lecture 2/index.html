Uniformed (Blind) Search Strategies

  • Uninformed strategies
    • 不知道任务的特定启发知识
    • 通过一个系统的方式去生成子node
    • 知道这个子node是不是目标node
    • 不知道非目标node的时候是不是比另一个node好
      • UCS除外，但是这是基于一个heuristic knowledge
  • 5 uniformed search strategeis
    • Breadth-first
    • Depth-limited
    • Uninform-cost
    • Iterative deepening
    • Depth-first

BFS

  • 展开最浅且未展开的node
  • 在fringe list的最后面添加子node
  • Solution Path通过**母节点 **找到最上的节点
  • 最浅的节点不是最优解
  • ![image-20210310143826304](/Users/michaellu/Library/Application Support/typora-user-images/image-20210310143826304.png)

UCS Uniform Cost Search

  • from least-cost to most cost

  • 因为在BFS中根本不会去考量step cost。所有的节点都是系统性的在每一层中扩展，从最左到最右

  • BFS是寻找最浅的目标节点，但是这可能并不不是最优的解法

  • UCS会考虑step cost

    • Expand the least-cost unexpanded node
      • node n with eht lowest paht cost g(n) from the initial state
    • 在fringe中插入节点，从root node开始增加path cost

  • 在每一个expand之后的node都需要去检查是不是goal node

UCS and BFS

  • g(n) is the cost from the root node to a given node
  • step cost is the cost between 2 nodes
  • Some definitions:
    • step cost is 1
    • step cost is the same - generalization of case 1
    • g(n) = depth(n)
    • g(n) is the same at each level and increasing as the level increases

DFS

  • Expand the deepest unexpanded node
  • insert successors at the front of the fringe
  • it’s not complete, 无限深度的空间时候会失效
  • not optimal, 可能比最优解需要的时间更长, 哪怕step cost是1
  • time complexity: O(b^m)
  • space : O(bm), 线性的, 非常好, 因为是深度优先搜索，在有边的情况下是一直遍历, 所以是线性情况

Depth Limited Search

是DFs加上了深度限制 l

  • complete： Yes, 因为限制了搜索深度
  • optimal：no
  • time：O(b^1)
  • space: O(bl)

但是这涉及到一个问题，如何选择一个不错的限制呢？

很多时候我们在解决问题之前是不知道state space的diameter的。

Iterative Deepening Search (IDS)

尝试在所有可能的depth limited search中应用DFS

  • 在深度从0到无限的时候应用dfs算法
  • 尝试去整合DFS和BFS的优势
    • DFS的低内存和BFS的完整性(completeness if b is finite and optimality if step costs are the same by reapted DFS searches with increased depth)
    • 接近root node的会被expand multiple times，但是这些相关的nodes数量很小，多数的nodes是靠近leaf nodes的，并不是root nodes
  • Complete: As BFS
  • Optimal: yes, as BFS
    • DFS is not optimal as the depth goes infinity
  • Time: O(b^d)
  • Space: O(bd), as DFS, linear
  • 如果拿IDS和DFS对比的话，哪里可以优化一下：
    • completeness和optimality

Informed vs Uninformed Search

  • Search strategies defines the order of node expansion
    • Uninformed
      • 系统性的展开节点
        • 就好像BFS一层一层，从左边到右边展开一样
      • 能知道一个给定节点是目标节点还是非目标节点
      • 不能对比两个非目标节点
        • do not know if one non-goal node is **better ** than another one based on heuristic knowlegde
      • 在寻找solution的时候一般是低效的
    • Informed
      • 可以对比非目标节点和目标节点
        • 知道非目标节点是不是比另一个更好
      • 更加高效

Greedy Search

  • 使用h value作为evalution function
  • h(n) 表示的estimated cost from n to a goal node
  • 展开拥有最小h的节点
    • 则表示的是到目标节点最近的路径
    • 所以Greedy Search 最小化h
  • Complete: Yes（当空间M是有限的时候）
    • NO, 当空间是无限的时候
  • Optimal: No
  • Time: O(b^m)
    • 但是一个好的试探算法可以显著提升
  • Space: O(B^m) 每一个节点都需要保留在内存中
